# 🛠 子供向けスクラム道場：開発プロジェクト戦略設計図

「プログラミングを教える」のではなく、**「自分たちの力でシステムを制御する思考」**をインストールするための戦略的設計図。

## 1. チームの役割定義
* **子供たち（プロダクトオーナー / スクラムマスター）**
    * 何をどう面白くするかを決める意志決定者。
    * プロダクトの責任を持ち、改善点を定義する。
* **あなた（デベロッパー / ファシリテーター）**
    * 子供たちの仕様を現実（コード）に落とし込む実装者。
    * 開発プロセスが停滞しないようスクラムを回す専門家。
* **AI**
    * 爆速実装を支える最強の共同開発パートナー。

## 2. スクラムの4ステップ・サイクル
各サイクルを15〜30分で回し、「作る→振り返る→改善する」という本物の開発フローを体感させる。

1. **スプリント計画 (Planning)**
    * 何が「面白そうか」をチームで握る。
2. **仕様定義 (Simulating)**
    * 各自が担当する領域の「仕様」を設計・図解する。
    * 各個人の成果物（仕様）としてホワイトボード等に書き出す。
3. **仕様レビュー (Review)**
    * 仕様を突き合わせ、矛盾やリスクをチームでつぶす。
4. **デプロイ (Deployment)**
    * あなたが各仕様をマージしブラウザで統合実行する。
5. **改善探索 (Retrospective & Next Planning)**
    * 動いたものを触り、「ここが不便だ」「もっとこうしたい」という改善案を自ら出し合い、次のIssueとしてバックログに積む。

## Projects Overview

本リポジトリの各サンプルは、ワークショップでの「自由に作って試す楽しさ」と「見た目の派手さ（プレゼン性）」を両立するために設計されています。コードの細部というより、どこまで自由に改変して良いかを判断し、短いスプリントで改善を回す体験を重視します。

- **BrickBreaker/** — 最小限のブロック崩しスケルトン。
    - 役割: すばやく手を入れられる出発点。自由にルールや演出を追加して「どこまで派手にできるか」を試すのに最適です。

- **SideScrollerShooter/** — シンプルな横スクロールシューティングの骨組み。
    - 役割: 敵パターンや弾のエフェクトを増やして見た目を派手にする実験台。短いイテレーションでバランスと演出（パーティクル・爆発）を調整します。

- **Platformer/** — 一部機能が揃った横スクロールアクション。
    - 役割: ゲームプレイ性と表現のバランスを学ぶ教材。レベル設計やアニメーション追加で「遊びやすさ」と「見栄え」を同時に改善します。

- **Wizardly/** — 高完成度の疑似3D一人称RPG。
    - 役割: 完成度が高いため、基本機能はそのままに“パラメータ調整”“見た目の演出強化（ライト／UI）”“ゲームバランス調整”を主体に学びます。モジュール分割（`config.js` / `mapData.js` / `characters.js`）を使って安全に改修する練習にも最適です。

各プロジェクトとも「まず小さく動くものを作る」「見た目で魅せる改善を短時間で試す」「結果をプレイテストして次の改善を決める」というサイクルを回すことが狙いです。ワークショップ向けに短いタスク例やプレイテストチェックリストを追加できます。希望があれば作成します。
- **SideScrollerShooter/**: 横スクロールシューティングのサンプル。敵生成・弾の管理・スコアリングの実装例が含まれます。
- **Templates/**: ワークショップで使うテンプレート類（`Backlog.md`、`仕様定義.md`、`振り返り.md`）を格納しています。
- **Wizardly/**: 疑似3D 一人称ダンジョン探索RPGプロジェクト。`config.js` / `mapData.js` / `characters.js` に分割されたゲームロジックと、キャンバス描画・戦闘システムを含みます。
