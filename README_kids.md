# 自由研究: ゲームづくりプロジェクト（やさしい説明）

## このプロジェクトで大事にしていること
このプロジェクトのねらいは、
みんながゲームづくりを通して
「自分で考える」「ためす」「みんなでよくする」ことを体験することです。

むずかしい言葉を覚えることより、
まずはやってみて、気づいて、直していくことを大切にします。

ワークショップでは、
みんながアイデアを出したり、画面や操作を考えたり、
遊んでチェックしたりします。

コードを書く作業や細かい技術の判断は、
先生やサポートする人が担当します。

## ゴール
- 問題を見つけて、直して、また試せるようになる
- アイデアを絵や言葉で分かりやすく伝えられるようになる
- 役わり分担をして、協力して作れるようになる

## 進め方（スクラムのしくみ）
スクラムは、
「短い作業」と「ふりかえり」をくり返して、
少しずつよくしていくチームの進め方です。

このプロジェクトでは、次の流れで進めます。

1. まず計画する
- 「何をやってみるか」をみんなで決めます（記録は先生）

2. 作る内容を決める
- どんな画面にするか、どんな操作にするかを、絵や言葉で決めます

3. 内容を確認する
- 遊んだときに困りそうなところを話し合います
- むずかしい技術のところは先生が手伝います

4. 動かしてたしかめる
- 先生が子どもの案をもとに実装して、ブラウザで動かします
- みんなで遊んで「ここがいい」「ここを直したい」を見つけます

5. ふりかえりと記録
- 感想や次に直したいことを出して、次のチャレンジを決めます

このサイクルを短く回すと、
「できた」が増えて、学びも増えていきます。

## 身につく力
- 順番に考える力
- 問題を見つけて直す力
- 伝える力（デザインや説明）
- 観察して意見を伝える力
- 協力して作る力
- 小さな成功を重ねる力
- デジタルのしくみを知る力

先生は技術を担当し、
みんなの考えがちゃんと形になるように支えます。

## 成果の見せ方・公開方法
みんなが作ったゲームは、ブラウザで遊べる形で公開します。
URLを共有したり、発表会で遊んでもらったりして、
「作れた!」という気持ちにつなげます。

ワークショップでは、
「どこを直すか（Issue）」や
「どう直したか（PR）」の記録も、先生がまとめて公開します。
このページを見ると、
「どんな話し合いをして、どうよくしたか」をふりかえれます。

  - Issue一覧: https://github.com/scrumrecreation/ScrumRecreation-template/issues

## このゲームを自分のパソコンで改造する方法
コードに興味が出てきた人だけ、ワークショップのあとでこの手順を使って自分のパソコンで試せるようにしています。
ここから先は「興味がある人向けの発展編」として読んでください。

### 1. VS Code（ブイエスコード）って何？
VS Codeは、プログラムを書くための無料アプリです。
メモ帳より見やすく、ファイルを整理しやすいので、このプロジェクトでも使いやすいです。

### 2. VS Codeのインストール
1. 公式サイトを開く: https://code.visualstudio.com/
2. 自分のOS（Windows / macOS / Linux）向けのボタンを押してダウンロード
3. ダウンロードしたファイルを開いて、画面の案内どおりにインストール

### 3. まずは ZIP でダウンロード（おすすめ）
1. GitHubのこのリポジトリページを開く
2. 緑の `Code` ボタンを押す
3. `Download ZIP` を押して保存する
4. ダウンロードしたZIPを解凍（展開）する

### 4. VS Codeでフォルダを開く（編集するとき）
1. VS Codeを開く
2. メニューの `ファイル` → `フォルダーを開く` を選ぶ
3. さっき解凍したフォルダを選ぶ
4. 左のエクスプローラーでファイル（例: `BrickBreaker/index.html`）を開く

### 5. ゲームをブラウザで開く
基本は「開きたいプロジェクトの `index.html`（または `*.html`）」をブラウザで開きます。

- 例（BrickBreakerの場合）: `BrickBreaker/index.html`

- Linuxなら、次のコマンドでも開けます。

```bash
xdg-open BrickBreaker/index.html
```

ほかのプロジェクトを開くときは、最後のパスだけ入れ替えてください。

### 6. Gitで取得する方法（オプション）
更新を取り込みたい人は、Gitで取得してもOKです。
