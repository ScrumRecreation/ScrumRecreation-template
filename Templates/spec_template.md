1) いつ動く？（TRIGGER）
- 短く書く：いつこの機能が始まるか。
- 例：ボールがブロックに当たったとき、プレイヤーがスペースキーを押したとき
- 記入欄：
```
（ここに記入してください）
```

2) 何をする？（ACTION）
- 短く書く：何が起こるか（画面で見えること・ゲームの中で起きること）。
- 例：スコアが＋10、ブロックが消える、ボールが速くなる
- 記入欄：
```
（ここに記入してください）
```

3) どんな条件がある？（CONDITION）
- 別の場合や例外を箇条書きで。できれば3つ以内にまとめる。
- 例：ボールが速すぎると効果なし、残りライフが0のときは無効
- 記入欄：
```
（ここに記入してください）
```

4) どう見える？（見た目／UI）
- 画面にどんな表示が出るか。色・文字・音などをわかりやすく書く。
- 例：画面右上に「+10」と黄色で表示、打音（pop.wav）を鳴らす
- 記入欄：
```
（ここに記入してください）
```

5) プレイヤーの操作（INPUT）
- プレイヤーが何をするか（キー、クリック、マウスなど）。
- 例：矢印キーでパドルを動かす、スペースでショット
- 記入欄：
```
（ここに記入してください）
```

6) できたとみなす条件（Acceptance Criteria）
- このチェックが全部OKなら完成とする。テストで確かめられるように短く書く。
- 例：ブロックに当たるとスコアが正しく10増えること、表示が1秒で消えること
- 記入欄：
```
（ここに記入してください）
```

7) テスト手順（誰でもできる）
- 1から5の短い手順でテストできるように。3〜5ステップ推奨。
- 例：
    1. ゲームを開く
    2. ブロックにボールを当てる
    3. 右上のスコアが+10されるか確認する
- 記入欄：
```
1. 
2. 
3. 
```


8) 備考（メモ）
- 追加で伝えたいことがあれば短く書く。
- 記入欄：
```
（ここに記入してください）
```
